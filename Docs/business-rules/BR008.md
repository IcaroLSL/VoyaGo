# BR006: busca de roteiros de outros usuários
## Regra de Negócio

Todo roteiro criado por um usuário deve ter a opção de ser buscado por outros usuários, respeitando as configurações de privacidade do roteiro. O sistema deve garantir que os roteiros públicos sejam visíveis para todos os usuários, enquanto os roteiros privados só sejam acessíveis para o usuário criador e para os usuários autorizados pelo criador. O sistema deve validar a permissão de visualização no momento da requisição de visualização do roteiro, retornando mensagens de erro e status code apropriados para casos de acesso negado ou roteiro inexistente, e permitindo a visualização para casos de acesso permitido.

## Especificação BDD

### Feature: Busca de roteiros de outros usuários

**Como** sistema de busca de roteiros de viagens
**Quero** permitir que usuários busquem roteiros criados por outros usuários
**Para** garantir que os usuários possam encontrar roteiros públicos e acessar roteiros privados compartilhados com eles, mantendo a privacidade dos dados e a integridade do sistema

---

### Cenário 1: Usuário busca roteiro

**Dado** um usuário devidamente logado no sistema com access token válido
**Quando** o usuário busca um roteiro público de outro usuário
**Então** o sistema deve validar a autenticação do usuário e permitir acesso à API na rota de busca, retornando o roteiro público e garantindo que o roteiro seja visível apenas para usuários autorizados

### Cenário 2: Usuário busca roteiro anteriormente favoritado e desfavorita o roteiro

**Dado** um usuário devidamente logado no sistema com access token válido e um roteiro que o usuário já favoritou anteriormente
**Quando** o usuário visualiza o roteiro favoritado e opta por desfavoritar-lo
**Então** o sistema deve validar a autenticação do usuário e permitir acesso à API na rota de desfavoritar, removendo o roteiro da lista de favoritos do usuário e garantindo que o roteiro desfavoritado não seja mais visível na lista de favoritos do usuário

### Cenário 3: Buscar roteiros por localização atual

**Dado** que o usuário está em "São Paulo, SP"  
**E** deseja encontrar atividades próximas  
**Quando** o sistema busca roteiros disponíveis  
**Então** deve retornar roteiros e atividades em um raio de até 50km  
**E** as atividades devem estar ordenadas por proximidade

### Cenário 4: Buscar roteiros por destino específico

**Dado** que o usuário deseja viajar para "Florianópolis, SC"  
**E** informou as datas de 10/03/2026 a 15/03/2026  
**Quando** o sistema busca roteiros disponíveis  
**Então** deve retornar roteiros compatíveis com 5 dias de viagem  
**E** as atividades devem considerar o período informado

### Cenário 5: Usuário faz uma busca por roteiro e ccomeça a adicionar filtros, mas não encontra atividades que atendam a todos os filtros

**Dado** um usuário devidamente logado no sistema com access token válido e um roteiro que o usuário está montando
**Quando** o usuário aplica múltiplos filtros (categoria, preço, avaliação) e não encontra atividades que atendam a todos os filtros
**Então** o sistema deve retornar uma mensagem informando que não foram encontradas atividades que atendam a todos os filtros aplicados, sugerindo ao usuário revisar os filtros ou expandir os critérios de busca para encontrar atividades compatíveis com suas preferências

### Cenário 6: Usuário faz uma busca por roteiro e adiciona filtros, mas encontra atividades que atendam a todos os filtros

**Dado** um usuário devidamente logado no sistema com access token válido e um roteiro que o usuário está montando
**Quando** o usuário aplica múltiplos filtros (categoria, preço, avaliação) e encontra atividades que atendam a todos os filtros
**Então** o sistema deve retornar as atividades que atendam a todos os filtros aplicados, permitindo que o usuário adicione as atividades ao seu roteiro e continue personalizando seu roteiro de viagem de acordo com suas preferências

### Cenário 7: Usuário busca roteiro e api está fora do ar

**Dado** um usuário devidamente logado no sistema com access token válido
**Quando** o usuário busca um roteiro de outro usuário e a API de busca está fora do ar
**Então** o sistema deve retornar uma mensagem de erro informando que a busca de roteiros está temporariamente indisponível, sugerindo ao usuário tentar novamente mais tarde e garantindo que o sistema lide com a falha sem comprometer a experiência do usuário ou a estabilidade do sistema

### Cenário 8: Usuário busca roteiro inexistente 

**Dado** um usuário devidamente logado no sistema com access token válido
**Quando** o usuário busca um roteiro que não existe no sistema
**Então** o sistema deve retornar uma mensagem de erro informando que o roteiro não foi encontrado, sugerindo ao usuário revisar os critérios de busca ou tentar novamente com um roteiro diferente, e garantindo que o sistema lide com a situação de forma adequada sem comprometer a experiência do usuário ou a estabilidade do sistema

### Cenário 9: Usuário busca roteiro privado sem permissão

**Dado** um usuário devidamente logado no sistema com access token válido
**Quando** o usuário busca um roteiro privado de outro usuário para o qual não tem permissão de acesso
**Então** o sistema deve retornar uma mensagem de erro informando que não foi possivel indenficar o roteiro ou que o roteiro não existe, sugerindo ao usuário revisar os critérios de busca ou tentar novamente com um roteiro diferente, e garantindo que o sistema lide com a situação de forma adequada sem comprometer a experiência do usuário ou a estabilidade do sistema

### Cenário 10: Usuário busca roteiro privado com token expirado

**Dado** um usuário devidamente logado no sistema com access token expirado
**Quando** o usuário busca um roteiro privado de outro usuário para o qual tem permissão de acesso, mas o token de acesso do usuário expirou
**Então** o sistema deve retornar uma mensagem de erro informando que a sessão do usuário expirou, sugerindo ao usuário fazer login novamente para obter um novo token de acesso, e garantindo que o sistema lide com a situação de forma adequada sem comprometer a experiência do usuário ou a estabilidade do sistema retornando status code 401(Unauthorized) para indicar que a autenticação é necessária

### Cenário 11: Usuário busca roteiro mas se encontra sem internet

**Dado** um usuário devidamente logado no sistema com access token válido
**Quando** o usuário busca um roteiro de outro usuário, mas o dispositivo do usuário está sem conexão com a internet
**Então** o sistema deve retornar uma mensagem de erro informando que a busca de roteiros não pode ser realizada devido à falta de conexão com a internet, sugerindo ao usuário verificar sua conexão e tentar novamente
