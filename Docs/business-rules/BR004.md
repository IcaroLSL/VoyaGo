# BR002: Grupos de Viagem Colaborativos

## Regra de Negócio

O sistema deve permitir que usuários criem grupos de viagem, convidem participantes e colaborem na criação e edição de roteiros compartilhados, mantendo controle de permissões e histórico de alterações.

## Especificação BDD

### Feature: Grupos de viagem colaborativos

**Como** viajante organizador  
**Quero** criar um grupo de viagem e convidar pessoas  
**Para** planejar roteiros de forma colaborativa com amigos e familiares

---

### Cenário 1: Criar grupo de viagem

**Dado** que o usuário está autenticado no sistema  
**E** deseja organizar uma viagem em grupo  
**Quando** cria um novo grupo com nome "Viagem Nordeste 2026"  
**Então** o grupo deve ser criado com sucesso  
**E** o usuário deve ser definido como ADMINISTRADOR do grupo  
**E** o grupo deve receber um código único de convite

### Cenário 2: Convidar participantes por código

**Dado** que o usuário é ADMINISTRADOR do grupo "Viagem Nordeste 2026"  
**E** possui o código de convite "ABC1234" (BASE62)  
**Quando** compartilha o código com um amigo  
**E** o amigo acessa o link de convite  
**Então** o amigo deve ser adicionado ao grupo como MEMBRO  
**E** deve visualizar os roteiros existentes do grupo

### Cenário 3: Convidar participantes por e-mail

**Dado** que o usuário é ADMINISTRADOR do grupo  
**E** deseja convidar "joao@email.com"  
**Quando** envia convite por e-mail  
**Então** o sistema deve enviar notificação ao destinatário  
**E** o convite deve ficar com status PENDENTE  
**E** o convite deve expirar em 7 dias se não aceito

### Cenário 4: Membro visualiza roteiros do grupo

**Dado** que o usuário é MEMBRO do grupo "Viagem Nordeste 2026"  
**E** o grupo possui 2 roteiros cadastrados  
**Quando** acessa a área do grupo  
**Então** deve visualizar todos os roteiros do grupo  
**E** deve ver quem criou cada roteiro  
**E** deve ver a última atualização de cada roteiro

### Cenário 5: Membro cria roteiro no grupo

**Dado** que o usuário é MEMBRO do grupo "Viagem Nordeste 2026"  
**E** possui permissão de criação ativa  
**Quando** cria um novo roteiro "Recife - 3 dias"  
**Então** o roteiro deve ser vinculado ao grupo  
**E** todos os membros devem poder visualizar o roteiro  
**E** o criador deve ser registrado no histórico

### Cenário 6: Membro edita roteiro existente

**Dado** que o usuário é MEMBRO do grupo  
**E** existe um roteiro "Salvador - 5 dias" no grupo  
**Quando** adiciona uma atividade "Visita ao Pelourinho"  
**Então** a alteração deve ser salva no roteiro  
**E** deve registrar no histórico quem fez a alteração  
**E** os demais membros devem ser notificados da mudança

### Cenário 7: Conflito de edição simultânea [OU NÃO PERMITIR DUAS PESSOAS EDITAREM AO MESMO TEMPO] ou ignorar esse erro é pra empresa grande

**Dado** que dois membros estão editando o mesmo roteiro  
**E** ambos alteram a atividade do dia 1 às 10:00  
**Quando** o segundo usuário tenta salvar  
**Então** deve exibir aviso "CONFLITO_EDICAO"  
**E** deve mostrar as duas versões lado a lado  
**E** deve permitir que o usuário escolha qual manter ou mesclar

### Cenário 8: Administrador remove participante

**Dado** que o usuário é ADMINISTRADOR do grupo  
**E** deseja remover o membro "Maria"  
**Quando** executa a remoção  
**Então** Maria deve perder acesso ao grupo  
**E** os roteiros criados por Maria devem permanecer no grupo  
**E** Maria deve ser notificada da remoção

### Cenário 9: Membro sai do grupo voluntariamente

**Dado** que o usuário é MEMBRO do grupo  
**E** deseja sair do grupo  
**Quando** confirma a saída  
**Então** deve perder acesso aos roteiros do grupo  
**E** os roteiros que criou devem permanecer no grupo  
**E** os demais membros devem ser notificados

### Cenário 10: Transferir administração do grupo [OU TRANSFERIR PARA O MEMBRO MAIS VELHO]

**Dado** que o usuário é o único ADMINISTRADOR do grupo  
**E** deseja sair do grupo  
**Quando** tenta sair sem transferir a administração  
**Então** deve exibir erro "ADMIN_OBRIGATORIO"  
**E** deve solicitar que transfira a administração para outro membro

## Regras de Papéis

```
ADMINISTRADOR: Controle total do grupo (criar, editar, excluir, gerenciar membros)
EDITOR:        Pode criar e editar roteiros, mas não gerencia membros
MEMBRO:        Pode visualizar e criar roteiros, mas não editar os de outros
CONVIDADO:     Apenas visualização (modo somente leitura)
```

## Regras de Convite

```
Código de convite: Formato VNG-{ANO}-{6 caracteres alfanuméricos}
Validade padrão: 7 dias
Máximo de usos por código: Ilimitado (configurável pelo admin)
Convite por e-mail: Expira em 7 dias
Reenvio de convite: Máximo 3 vezes por destinatário
```

## Regras de Grupo

```
Nome do grupo: 3 a 50 caracteres
Descrição: Até 500 caracteres (opcional)
Máximo de membros: 50 participantes por grupo
Máximo de roteiros por grupo: 20 roteiros
Máximo de grupos por usuário: 10 grupos (como admin) / Ilimitado (como membro)
```

## Regras de Histórico

```
Retenção de histórico: 90 dias
Informações registradas:
  - Data/hora da alteração
  - Usuário responsável
  - Tipo de alteração (criação, edição, exclusão)
  - Descrição resumida da mudança
  - Snapshot do estado anterior (para roteiros)
```

## Regras de Notificação

```
Notificar membros quando:
  - Novo roteiro criado no grupo
  - Roteiro editado (exceto alterações próprias)
  - Novo membro adicionado
  - Membro removido ou saiu
  - Convite recebido
  - Convite aceito
  
Canais de notificação:
  - Push notification (app)
  - E-mail (configurável)
  - In-app notification
```

## ⚠️ Exceções

```java
GrupoNaoEncontradoException
  - código: "GRUPO_NAO_ENCONTRADO"
  - mensagem: "O grupo solicitado não foi encontrado"
  - httpStatus: 404 (Not Found)

PermissaoNegadaException
  - código: "PERMISSAO_NEGADA"
  - mensagem: "Você não tem permissão para realizar esta ação"
  - httpStatus: 403 (Forbidden)

ConviteExpiradoException
  - código: "CONVITE_EXPIRADO"
  - mensagem: "Este convite expirou. Solicite um novo convite ao administrador"
  - httpStatus: 410 (Gone)

ConviteInvalidoException
  - código: "CONVITE_INVALIDO"
  - mensagem: "O código de convite informado é inválido"
  - httpStatus: 400 (Bad Request)

LimiteMembrosExcedidoException
  - código: "LIMITE_MEMBROS_EXCEDIDO"
  - mensagem: "O grupo atingiu o limite máximo de 50 membros"
  - httpStatus: 400 (Bad Request)

LimiteGruposExcedidoException
  - código: "LIMITE_GRUPOS_EXCEDIDO"
  - mensagem: "Você atingiu o limite máximo de 10 grupos como administrador"
  - httpStatus: 400 (Bad Request)

AdminObrigatorioException
  - código: "ADMIN_OBRIGATORIO"
  - mensagem: "O grupo precisa de pelo menos um administrador. Transfira a administração antes de sair"
  - httpStatus: 400 (Bad Request)

ConflitoEdicaoException
  - código: "CONFLITO_EDICAO"
  - mensagem: "Outro usuário editou este roteiro. Revise as alterações antes de salvar"
  - httpStatus: 409 (Conflict)

MembroJaExisteException
  - código: "MEMBRO_JA_EXISTE"
  - mensagem: "Este usuário já é membro do grupo"
  - httpStatus: 409 (Conflict)
```
