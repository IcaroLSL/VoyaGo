# BR001: Montagem e Busca de Roteiros de Viagem Personalizados

## Regra de Negócio

O sistema deve permitir que usuários busquem ou montem roteiros de viagem personalizados baseados em sua localização atual ou destino desejado, organizando atividades por dia, horário, categoria, faixa de preço e avaliações.

## Especificação BDD

### Feature: Busca e montagem de roteiros de viagem personalizados

**Como** viajante  
**Quero** buscar ou montar um roteiro de viagem baseado na minha localização ou destino  
**Para** planejar minhas atividades de forma organizada e personalizada

---

### Cenário 1: Buscar roteiros por localização atual

**Dado** que o usuário está em "São Paulo, SP"  
**E** deseja encontrar atividades próximas  
**Quando** o sistema busca roteiros disponíveis  
**Então** deve retornar roteiros e atividades em um raio de até 50km  
**E** as atividades devem estar ordenadas por proximidade

### Cenário 2: Buscar roteiros por destino específico

**Dado** que o usuário deseja viajar para "Florianópolis, SC"  
**E** informou as datas de 10/03/2026 a 15/03/2026  
**Quando** o sistema busca roteiros disponíveis  
**Então** deve retornar roteiros compatíveis com 5 dias de viagem  
**E** as atividades devem considerar o período informado

### Cenário 3: Montar roteiro com atividades por categoria

**Dado** que o usuário está montando um roteiro para "Rio de Janeiro"  
**E** selecionou as categorias "PRAIA" e "RESTAURANTE"  
**Quando** adiciona atividades ao dia 1  
**Então** deve exibir apenas atividades das categorias selecionadas  
**E** deve permitir definir horário para cada atividade

### Cenário 4: Filtrar atividades por faixa de preço

**Dado** que o usuário está montando um roteiro  
**E** definiu orçamento máximo de 1 cifrão por atividade  
**Quando** o sistema lista atividades disponíveis  
**Então** deve exibir apenas atividades com que estejam entre 1 cifrao 
**E** deve mostrar o preço médio de cada atividade

### Cenário 5: Ordenar atividades por avaliação

**Dado** que o usuário está buscando restaurantes em "Gramado, RS"  
**E** deseja ver os mais bem avaliados  
**Quando** ordena por avaliação  
**Então** deve exibir atividades ordenadas por nota (maior para menor)  
**E** deve mostrar a nota média e quantidade de avaliações

### Cenário 6: Validar conflito de horários no roteiro

**Dado** que o usuário adicionou "Visita ao Museu" das 10:00 às 12:00  
**E** tenta adicionar "Almoço no Restaurante X" das 11:30 às 13:00  
**Quando** o sistema valida o roteiro  
**Então** deve retornar erro "CONFLITO_HORARIO"  
**E** deve informar "Existe sobreposição de horários entre atividades"

### Cenário 7: Sugerir atividades baseadas em preferências

**Dado** que o usuário tem histórico de preferência por "NATUREZA"  
**E** está montando roteiro para "Bonito, MS"  
**Quando** o sistema sugere atividades  
**Então** deve priorizar atividades da categoria "NATUREZA"  
**E** deve considerar avaliações acima de 4.0

## Regras de Categorias

```
RESTAURANTE:       Estabelecimentos gastronômicos
MUSEU:             Museus, galerias e centros culturais
ATIVIDADE_AR_LIVRE: Trilhas, esportes, passeios ao ar livre
PRAIA:             Praias e atividades costeiras
PARQUE:            Parques naturais e urbanos
VIDA_NOTURNA:      Bares, baladas, shows
COMPRAS:           Shoppings, feiras, mercados
HISTORICO:         Monumentos, sítios históricos
```

## Regras de Faixa de Preço REVER DEPOIS DE VER A API

```
GRATUITO:    gratuíto
ECONOMICO:   usar 1 cifrão $
MODERADO:    usar 2 cifrão $
PREMIUM:     usar 3 cifrão $
LUXO:        usar 4 cifrão $
```

## Regras de Avaliação

```
Escala: 1.0 a 5.0 estrelas
Mínimo para destaque: 4.0 estrelas
Mínimo de avaliações para ranking: 10 avaliações
Peso recente: Avaliações dos últimos 6 meses têm peso 1.5x
```

## Regras de Horário

```
MANHA:       06:00 - 12:00
TARDE:       12:00 - 18:00
NOITE:       18:00 - 00:00
MADRUGADA:   00:00 - 06:00

Duração mínima de atividade: 30 minutos
Duração máxima de atividade: 8 horas
Intervalo mínimo entre atividades: 15 minutos (deslocamento)
```

## Regras de Geolocalização

```
Raio padrão de busca: 50km
Raio mínimo: 1km
Raio máximo: 200km
Precisão de localização: GPS ou endereço informado
```

## ⚠️ Exceções

```java
ConflitoHorarioException
  - código: "CONFLITO_HORARIO"
  - mensagem: "Existe sobreposição de horários entre atividades"
  - httpStatus: 400 (Bad Request)

LocalizacaoNaoEncontradaException
  - código: "LOCALIZACAO_NAO_ENCONTRADA"
  - mensagem: "Não foi possível determinar a localização informada"
  - httpStatus: 400 (Bad Request)

AtividadeIndisponivelException
  - código: "ATIVIDADE_INDISPONIVEL"
  - mensagem: "A atividade não está disponível no horário selecionado"
  - httpStatus: 409 (Conflict)

RoteiroInvalidoException
  - código: "ROTEIRO_INVALIDO"
  - mensagem: "O roteiro não atende aos requisitos mínimos"
  - httpStatus: 400 (Bad Request)

LimiteAtividadesExcedidoException
  - código: "LIMITE_ATIVIDADES_EXCEDIDO"
  - mensagem: "Número máximo de atividades por dia excedido (máx: 10)"
  - httpStatus: 400 (Bad Request)
```
