# BR006: Sistema de favoritar roteiros
## Regra de Negócio

Todo roteiro criado por um usuário deve ter a opção de ser favoritado por outros usuários. Um usuário pode marcar um roteiro como favorito, o que permite que ele seja facilmente acessado posteriormente na lista de roteiros favoritos. O sistema deve garantir que os roteiros favoritados sejam visíveis apenas para o usuário que os favoritou, mantendo a integridade dos dados e a privacidade do usuário.

## Especificação BDD

### Feature: Visualização de roteiros de outros usuários

**Como** sistema de favoritar roteiros de viagens
**Quero** permitir que usuários favoritem roteiros de outros usuários
**Para** garantir que os usuários possam marcar roteiros como favoritos para fácil acesso posterior, mantendo a privacidade dos dados e a integridade do sistema

---

### Cenário 1: Usuário visualiza roteiro público e favorita o roteiro

**Dado** um usuário devidamente logado no sistema com access token válido
**Quando** o usuário visualiza um roteiro público de outro usuário e opta por favoritar-lo
**Então** o sistema deve validar a autenticação do usuário e permitir acesso à API na rota de favoritar, adicionando o roteiro à lista de favoritos do usuário e garantindo que o roteiro favoritado seja visível apenas para o usuário que o favoritou

### Cenário 2: Usuário visualiza roteiro anteriormente favoritado e desfavorita o roteiro

**Dado** um usuário devidamente logado no sistema com access token válido e um roteiro que o usuário já favoritou anteriormente
**Quando** o usuário visualiza o roteiro favoritado e opta por desfavoritar-lo
**Então** o sistema deve validar a autenticação do usuário e permitir acesso à API na rota de desfavoritar, removendo o roteiro da lista de favoritos do usuário e garantindo que o roteiro desfavoritado não seja mais visível na lista de favoritos do usuário

### Cenário 3: Usuário tenta acessar um roteiro anteriormente publico 

**Dado** um roteiro com configuração de privacidade "PRIVADO" criado por outro usuário e compartilhado via link para o usuário que tenta visualizar
**Quando** o usuário logado acessa o link compartilhado e tenta visualizar o roteiro
**Então** o sistema deve validar a autenticação do usuário e permitir a visualização do roteiro, exibindo seus detalhes como destinos, permanências e tipo de roteiro e adicionando o usuario que acessou o link a allow list do roteiro para futuras visualizações

### Cenário 4: Usuario visualiza roteiro colaborativo compartilhado via link

**Dado** um roteiro com configuração de privacidade "COLABORATIVO" criado por outro usuário e compartilhado via link para o usuário que tenta visualizar
**Quando** o usuário logado acessa o link compartilhado e tenta visualizar o roteiro colaborativo
**Então** o sistema deve validar a autenticação do usuário e permitir a visualização do roteiro, exibindo seus detalhes como destinos, permanências e tipo de roteiro e adicionando o usuario que acessou o link a allow list do roteiro para futuras visualizações e edições do roteiro colaborativo

### Cenário 5: Usuário visualiza roteiro colaborativo compartilhado via allow list

**Dado** um roteiro com configuração de privacidade "COLABORATIVO" criado por outro usuário e compartilhado via allow list para o usuário que tenta visualizar
**Quando** o usuário logado tenta visualizar o roteiro colaborativo compartilhado via allow list
**Então** o sistema deve validar a autenticação do usuário e permitir a visualização do roteiro, exibindo seus detalhes como destinos, permanências e tipo de roteiro e permitindo que o usuário edite o roteiro colaborativo

### Cenário 6: Usuário visualiza roteiro privado de outro usuário sem permissão

**Dado** um roteiro com configuração de privacidade "PRIVADO" criado por outro usuário
**Quando** o usuário logado tenta visualizar o roteiro privado sem estar na allow list do roteiro
**Então** o sistema deve negar a visualização do roteiro e exibir uma mensagem de erro informando que o usuário não tem permissão para visualizar o roteiro  e status code 403 (Forbidden)

### Cenário 7: Usuário visualiza roteiro inexistente

**Dado** um roteiro que não existe ou foi deletado
**Quando** o usuário logado tenta visualizar o roteiro inexistente
**Então** o sistema deve negar a visualização do roteiro e exibir uma mensagem de erro informando que o roteiro não existe ou foi deletado e status code 404 (Not Found)

### Cenário 8: Usuário visualiza seu próprio roteiro privado

**Dado** um roteiro com configuração de privacidade "PRIVADO" criado pelo próprio usuário
**Quando** o usuário logado tenta visualizar seu próprio roteiro privado
**Então** o sistema deve permitir a visualização do roteiro, exibindo seus detalhes como destinos, permanências e tipo de roteiro

### Cenário 9: Usuário visualiza roteiro sem allow list ou link

**Dado** um roteiro com configuração de privacidade "PRIVADO" criado por outro usuário sem compartilhamento via allow list ou link
**Quando** o usuário logado tenta visualizar o roteiro privado sem compartilhamento
**Então** o sistema deve negar a visualização do roteiro e exibir uma mensagem de erro e status code 403 (Forbidden)

### Cenário 10: Usuário visualiza roteiro privado de outro usuário compartilhado via link com token expirado

**Dado** um roteiro com configuração de privacidade "PRIVADO" criado por outro usuário e compartilhado via link para o usuário que tenta visualizar, porém o token de acesso do link expirou
**Quando** o usuário logado acessa o link compartilhado e tenta visualizar o roteiro
**Então** o sistema deve validar a autenticação do usuário e negar a visualização do roteiro, exibindo uma mensagem de erro informando que o token de acesso expirou e status code 401 (Unauthorized)

### Cenário 11: Usuário visualiza roteiro colaborativo compartilhado via link com token expirado

**Dado** um roteiro com configuração de privacidade "COLABORATIVO" criado por outro usuário e compartilhado via link para o usuário que tenta visualizar, porém o token de acesso do link expirou
**Quando** o usuário logado acessa o link compartilhado e tenta visualizar o roteiro colaborativo
**Então** o sistema deve validar a autenticação do usuário e negar a visualização do roteiro, exibindo uma mensagem de erro informando que o token de acesso expirou e status code 401 (Unauthorized)
 
### Cenário 12: Usuário visualiza roteiro mas ocorre falha no servidor

**Dado** um roteiro criado por outro usuário
**Quando** o usuário logado tenta visualizar o roteiro e ocorre uma falha no servidor durante o processo de visualização
**Então** o sistema deve retornar uma resposta de erro indicando que ocorreu um erro interno no servidor durante o processo de visualização e status code 500 (Internal Server Error)
