# BR005: Sistema de Convites para Grupos de Viagem

## Regra de Negócio

O sistema deve permitir que usuários convidem outras pessoas para participar de grupos de viagem através de múltiplos canais: código de convite (Base62 com 7 dígitos), compartilhamento via WhatsApp e convite direto pelo aplicativo utilizando e-mail. O destinatário do convite deve receber notificação e poder aceitar ou recusar a participação no grupo.

## Especificação BDD

### Feature: Convites para grupos de viagem

**Como** organizador de viagem  
**Quero** convidar pessoas para meu grupo de viagem por diferentes canais  
**Para** facilitar a colaboração no planejamento do roteiro com amigos e familiares

---

### Cenário 1: Gerar código de convite Base62

**Dado** que o usuário é ADMINISTRADOR do grupo "Viagem Nordeste 2026"  
**Quando** solicita a geração de um código de convite  
**Então** o sistema deve gerar um código único em Base62 com 7 dígitos  
**E** o código deve ser válido por 7 dias (configurável)  
**E** o código deve ser exibido para compartilhamento

### Cenário 2: Validar formato do código Base62

**Dado** que o sistema precisa gerar um código de convite  
**Quando** o código é gerado  
**Então** deve conter exatamente 7 caracteres  
**E** deve utilizar apenas caracteres do alfabeto Base62 (0-9, a-z, A-Z)  
**E** deve ser único no sistema (verificação de colisão)

### Cenário 3: Convidar via código de convite

**Dado** que o usuário possui o código de convite "Ab3Cd7X"  
**E** o código está válido e não expirado  
**Quando** insere o código na área de "Entrar em Grupo"  
**Então** deve visualizar informações do grupo (nome, descrição, quantidade de membros)  
**E** deve poder aceitar ou recusar o convite  
**E** ao aceitar, deve ser adicionado como MEMBRO do grupo

### Cenário 4: Código de convite expirado

**Dado** que o usuário possui o código de convite "Xy7Zw2K"  
**E** o código expirou há mais de 24 horas  
**Quando** tenta utilizar o código  
**Então** deve exibir erro "CONVITE_EXPIRADO"  
**E** deve sugerir solicitar novo código ao administrador do grupo

### Cenário 5: Compartilhar convite via WhatsApp

**Dado** que o usuário é ADMINISTRADOR do grupo "Viagem Nordeste 2026"  
**E** possui código de convite ativo "Ab3Cd7X"  
**Quando** seleciona a opção "Compartilhar via WhatsApp"  
**Então** o sistema deve abrir o WhatsApp com mensagem pré-formatada  
**E** a mensagem deve conter o nome do grupo  
**E** a mensagem deve conter o link de convite com o código  
**E** a mensagem deve conter breve descrição do VoyaGo

### Cenário 6: Formato da mensagem WhatsApp

**Dado** que o administrador compartilha convite via WhatsApp  
**Quando** a mensagem é gerada  
**Então** deve seguir o formato:
```
🌎 VoyaGo - Planeje sua viagem!

Você foi convidado(a) para o grupo de viagem:
📍 {nome_do_grupo}

Clique no link para participar:
https://voyago.app/convite/{codigo}

Baixe o app: https://voyago.app/download
```

### Cenário 7: Convidar via aplicativo (por username)

**Dado** que o usuário é ADMINISTRADOR do grupo "Viagem Nordeste 2026"  
**E** deseja convidar "maria"  
**Quando** envia convite pelo aplicativo informando o username  
**Então** o sistema deve validar se o username está cadastrado no VoyaGo  
**E** se cadastrado, deve enviar notificação push para o usuário  
**E** se não cadastrado, deve enviar e-mail de convite com link para cadastro

### Cenário 8: Usuário cadastrado recebe notificação de convite

**Dado** que "maria" está cadastrada no VoyaGo  
**E** foi convidada para o grupo "Viagem Nordeste 2026"  
**Quando** o convite é enviado  
**Então** Maria deve receber notificação push no aplicativo  
**E** a notificação deve exibir "{nome_admin} convidou você para {nome_grupo}"  
**E** Maria deve visualizar o convite na área de notificações do app  
**E** o convite deve mostrar opções "ACEITAR" e "RECUSAR"

### Cenário 9: Aceitar convite via notificação

**Dado** que Maria recebeu convite para o grupo "Viagem Nordeste 2026"  
**E** o convite está com status PENDENTE  
**Quando** Maria clica em "ACEITAR"  
**Então** Maria deve ser adicionada ao grupo como MEMBRO  
**E** o status do convite deve mudar para ACEITO  
**E** o administrador deve ser notificado "Maria aceitou seu convite"  
**E** Maria deve ser redirecionada para a página do grupo

### Cenário 10: Recusar convite via notificação

**Dado** que Maria recebeu convite para o grupo "Viagem Nordeste 2026"  
**E** o convite está com status PENDENTE  
**Quando** Maria clica em "RECUSAR"  
**Então** o status do convite deve mudar para RECUSADO  
**E** Maria não deve ser adicionada ao grupo  
**E** o administrador deve ser notificado "Maria recusou seu convite"  
**E** o convite deve ser arquivado (não exibido novamente)

### Cenário 11: Usuário não cadastrado recebe convite por e-mail

**Dado** que "joao@email.com" NÃO está cadastrado no VoyaGo  
**E** foi convidado para o grupo "Viagem Nordeste 2026"  
**Quando** o convite é enviado  
**Então** João deve receber e-mail com convite  
**E** o e-mail deve conter link para cadastro + aceitação do convite  
**E** ao completar cadastro, João deve ser automaticamente adicionado ao grupo

### Cenário 12: Limite de convites pendentes por grupo

**Dado** que o grupo "Viagem Nordeste 2026" possui 20 convites pendentes  
**E** o limite máximo é 20 convites pendentes  
**Quando** o administrador tenta enviar novo convite  
**Então** deve exibir erro "LIMITE_CONVITES_PENDENTES"  
**E** deve sugerir cancelar convites antigos ou aguardar respostas

### Cenário 13: Reenviar convite

**Dado** que o convite para "maria@email.com" está PENDENTE há 5 dias  
**E** o administrador deseja reenviar  
**Quando** executa o reenvio  
**Então** deve renovar a validade do convite por mais 7 dias  
**E** deve enviar nova notificação/e-mail  
**E** deve registrar no histórico a quantidade de reenvios  
**E** máximo de 3 reenvios por convite

### Cenário 14: Cancelar convite pendente

**Dado** que existe convite PENDENTE para "joao@email.com"  
**E** o administrador deseja cancelar  
**Quando** executa o cancelamento  
**Então** o status do convite deve mudar para CANCELADO  
**E** o código de convite pessoal deve ser invalidado  
**E** João deve ser notificado do cancelamento (se cadastrado)

### Cenário 15: Usuário já é membro do grupo

**Dado** que Maria já é MEMBRO do grupo "Viagem Nordeste 2026"  
**E** o administrador tenta convidá-la novamente  
**Quando** envia o convite  
**Então** deve exibir erro "MEMBRO_JA_EXISTE"  
**E** não deve criar novo convite

## Regras de Código de Convite

```
Formato: Base62 (0-9, a-z, A-Z)
Tamanho: 7 caracteres
Exemplos válidos: "Ab3Cd7X", "9xYz2Kw", "mN4pQ8r"
Validade padrão: 1 ano
Regeneração: Admin pode gerar novo código a qualquer momento (exclui o código anterior)
Unicidade: Verificação de colisão obrigatória antes de persistir
```

## Regras de Convite por Canal

```
VIA CÓDIGO:
  - Qualquer pessoa com o código pode solicitar entrada
  - Requer conta no VoyaGo para aceitar
  - Código pode ser compartilhado livremente

VIA WHATSAPP:
  - Gera link com código embutido
  - Mensagem pré-formatada com informações do grupo
  - Deep link para abrir diretamente no app (se instalado)
  - Fallback para página web (se app não instalado)

VIA APLICATIVO (username):
  - Requer username válido do destinatário
  - Se usuário cadastrado: notificação push + in-app
  - Se não cadastrado: e-mail com link de cadastro
  - Convite pessoal e intransferível
  - Aceite/Recusa diretamente no app
```

## Regras de Notificação de Convite

```
PUSH NOTIFICATION:
  Título: "Convite para grupo de viagem"
  Corpo: "{nome_admin} convidou você para {nome_grupo}"
  Ação: Abre tela de aceite/recusa

IN-APP NOTIFICATION:
  Exibe na área de notificações
  Badge no ícone de notificações
  Persiste até ação do usuário

E-MAIL (usuário não cadastrado):
  Assunto: "Você foi convidado para uma viagem no VoyaGo!"
  Contém: Informações do grupo, quem convidou, link de cadastro
  Template responsivo para mobile
```

## Regras de Limites

```
Convites recusados (cooldown): 2 horas antes de poder convidar novamente
Códigos ativos por grupo: 1 código
Validade máxima: 1 ano
```

## Regras de Status do Convite

```
PENDENTE:    Convite enviado, aguardando resposta
ACEITO:      Usuário aceitou e foi adicionado ao grupo
RECUSADO:    Usuário recusou o convite
EXPIRADO:    Prazo de validade excedido sem resposta
CANCELADO:   Administrador cancelou o convite
```

## Algoritmo de Geração Base62

```java
public class GeradorCodigoService {
    
    private static final String BASE62_CHARS = 
        "0123456789ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz";
    private static final int TAMANHO_CODIGO = 7;
    private static final SecureRandom random = new SecureRandom();
    
    public CodigoBase62 gerarCodigo() {
        StringBuilder codigo = new StringBuilder(TAMANHO_CODIGO);
        for (int i = 0; i < TAMANHO_CODIGO; i++) {
            int index = random.nextInt(BASE62_CHARS.length());
            codigo.append(BASE62_CHARS.charAt(index));
        }
        return new CodigoBase62(codigo.toString());
    }
    
    public CodigoBase62 gerarCodigoUnico(ConviteRepositoryPort repository) {
        CodigoBase62 codigo;
        int tentativas = 0;
        do {
            if (tentativas++ > 10) {
                throw new GeracaoCodigoException("Falha ao gerar código único");
            }
            codigo = gerarCodigo();
        } while (repository.existeCodigo(codigo.valor()));
        return codigo;
    }
}
```

## ⚠️ Exceções

```java
ConviteNaoEncontradoException
  - código: "CONVITE_NAO_ENCONTRADO"
  - mensagem: "O convite solicitado não foi encontrado"
  - httpStatus: 404 (Not Found)

ConviteExpiradoException
  - código: "CONVITE_EXPIRADO"
  - mensagem: "Este convite expirou. Solicite um novo convite ao administrador"
  - httpStatus: 410 (Gone)

CodigoInvalidoException
  - código: "CODIGO_INVALIDO"
  - mensagem: "O código de convite informado é inválido ou não existe"
  - httpStatus: 400 (Bad Request)

LimiteConvitesPendentesException
  - código: "LIMITE_CONVITES_PENDENTES"
  - mensagem: "O grupo atingiu o limite máximo de 20 convites pendentes"
  - httpStatus: 400 (Bad Request)

LimiteReenviosException
  - código: "LIMITE_REENVIOS"
  - mensagem: "Este convite já foi reenviado o máximo de 3 vezes"
  - httpStatus: 400 (Bad Request)

MembroJaExisteException
  - código: "MEMBRO_JA_EXISTE"
  - mensagem: "Este usuário já é membro do grupo"
  - httpStatus: 409 (Conflict)

UsuarioBloqueadoException
  - código: "USUARIO_BLOQUEADO"
  - mensagem: "Você não pode entrar neste grupo"
  - httpStatus: 403 (Forbidden)

ConviteJaProcessadoException
  - código: "CONVITE_JA_PROCESSADO"
  - mensagem: "Este convite já foi aceito ou recusado"
  - httpStatus: 400 (Bad Request)

EmailInvalidoException
  - código: "EMAIL_INVALIDO"
  - mensagem: "O e-mail informado não é válido"
  - httpStatus: 400 (Bad Request)

GeracaoCodigoException
  - código: "ERRO_GERACAO_CODIGO"
  - mensagem: "Não foi possível gerar o código de convite. Tente novamente"
  - httpStatus: 500 (Internal Server Error)

CooldownConviteException
  - código: "COOLDOWN_CONVITE"
  - mensagem: "Este usuário recusou um convite recentemente. Aguarde 30 dias para convidar novamente"
  - httpStatus: 400 (Bad Request)
```
